System Blueprint

A closer look at the secure, resilient, and fully automated deployment infrastructure that runs this static application.

Staging Environment

A completely separate staging environment deployed automatically upon successful linting and testing of new commits. Configured using AWS S3 for hosting and fronted by a dedicated CloudFront distribution for caching and HTTPS enforcement. Perfect for pre-production quality check validations.

Blue-Green Routing

Two production S3 buckets (representing "Blue" and "Green" environments) are provisioned. A single production CloudFront distribution fronts these environments. The active environment is tracked in an AWS Systems Manager (SSM) Parameter. The deploy script switches the active origin without rebuilding, enabling instant promotions and rollbacks.

IAM Least-Privilege & OIDC

Security is maintained by avoiding long-lived credentials in GitHub. Instead, GitHub Actions establishes a secure trust relationship with AWS using OpenID Connect (OIDC) to temporarily assume an IAM role. The role is strictly scoped to deploy artifacts and update CloudFront invalidations.
